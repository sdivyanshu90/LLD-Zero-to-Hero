# Easy Problems

Focus area: OOP purity, SOLID adherence, basic design patterns, and no multithreading.

1. [Parking Lot](parking-lot/README.md): Truck occupies two adjacent spots, and split spots must merge again after departure.
2. [LRU Cache](lru-cache/README.md): O(1) `get()` and `put()` using a doubly linked list plus hash map.
3. [Logger Library](logger-library/README.md): Singleton logger with Chain of Responsibility for INFO, DEBUG, and ERROR routing.
4. [Vending Machine](vending-machine/README.md): Strict State Pattern with exact-change calculation.
5. [Tic-Tac-Toe](tic-tac-toe/README.md): N x N board with O(1) win detection via row, column, and diagonal counters.
6. [Library Management](library-management/README.md): Separate `Book` from `BookItem`; use Observer for reservations.
7. [Rule-Based Pricing Engine](rule-based-pricing-engine/README.md): Strategy Pattern with overlapping discount priority rules.
8. [Task Management](task-management/README.md): Composite Pattern for nested tasks and recursive completion percentage.
9. [Blackjack](blackjack/README.md): Dynamic Ace scoring without hardcoding hand values.
10. [File System](file-system/README.md): Composite Pattern with BFS and DFS search by size or extension.
