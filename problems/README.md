# Problems

This directory contains problem statements, constraints, assumptions, edge cases, and design guidance only.

## Rules

- Do not place Python code anywhere under `problems/`.
- Each problem should eventually have its own folder with a README.
- Problem READMEs should explain the business context, functional requirements, non-functional requirements, hidden traps, expected UML, and interview follow-up questions.
